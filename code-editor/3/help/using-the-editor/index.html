Using the Editor

This page provides a basic tutorial on using the HTML, CSS, and JavaScript editor. If you want to learn how to use the editors for other languages, click the 'How to Use' button in the editor you want to use.

The Editor Interface

There are several buttons above the editor, which do different things. They will be mentioned again in a later section. For now, let's look at the editor itself.

There are two sections of the editor's interface currently. The first one is the area where you type in code you want to run live. The second one is the output area, where you actually see the output, after you click or tap the "Run" button or press "Ctrl-Enter" on your keyboard.

If you have a wide screen (a screen width greater than or equal to 1200px), both the text editor and the output area will always be present on the left and right sides of the screen, respectively. If you have a narrow screen (a screen width less than 1200px), there will be buttons at the top right to switch between the text editor and the output area. If you don't know what your screen width is, use the button below to display it in a tooltip.

The Buttons

There are normally three buttons at the top left, which you can click or tap to perform certain actions. In the editors for the other languages, there is a fourth "How to Use" button, which opens a page similar to this one. The only difference is that the page tells you how to use that specific editor, NOT the HTML, CSS, and JavaScript editor.

Run

The first button, labeled by "Run", takes the code you typed into the text editor, interprets it, and shows the output. However, there is a 250-millisecond cooldown for this button, to prevent spamming.

Share

This next button, labeled by "Share", encodes the text in the editor, and uses it to create a link, which you can share to others. If it worked, you should see a tooltip right below the button, which as of now says "Link successfully copied!"

Clear Output

The third button, labeled by "Clear Output", clears the output for you. This is useful in cases where you run the code, and it gets a little too hectic.
